# Список из 1 000 000 элементов — выбор и сортировка

Тестовое задание (fullstack). Backend — **Express.js**, frontend — **React** (Vite + TypeScript).

## Возможности

**Левое окно «Доступные»** — все элементы, кроме выбранных:
- фильтрация по ID;
- инфинити-скролл (первая загрузка 20, далее порциями по 20);
- добавление новых элементов с произвольным новым ID.

**Правое окно «Выбранные»**:
- фильтрация по ID;
- сортировка через Drag & Drop (работает и для отфильтрованного списка);
- инфинити-скролл (20 + по 20);
- состояние выбора и сортировки хранится на сервере и переживает перезагрузку
  страницы (поиск не сохраняется).

**Общее:**
- результаты сортировки и выбора хранятся в памяти сервера (без БД);
- данные общие для всех посетителей, учётных записей нет;
- все запросы идут через **очередь с дедупликацией** — одно и то же значение не
  отправляется повторно;
- **батчинг**: добавление элементов отправляется раз в 10 сек, чтение и изменение
  данных — раз в 1 сек.

## Архитектура

```
server/   Express. In-memory store (store.js) + serialized write queue with
          de-dup (queue.js). Универсум = [1..1 000 000] ∪ добавленные ID.
client/   React + Vite + TypeScript. dnd-kit для Drag & Drop.
          requestQueue.ts — клиентская очередь с дедупликацией и батчингом.
```

### Ключевые решения

- **Левое окно** пагинируется на сервере (≈1 млн элементов нельзя держать
  на клиенте). Поиск — подстрочное совпадение по ID.
- **Правое окно** (выбранные) клиент держит целиком и нарезает по 20 при
  скролле — это даёт мгновенный Drag & Drop. При перезагрузке порядок
  восстанавливается с сервера (`GET /api/selected/order`).
- **Drag & Drop при фильтре**: переупорядочивание видимых строк переносится в
  полный порядок по «слотам» — невидимые элементы остаются на своих местах.
- **Очередь/дедуп/батчинг** на клиенте (`requestQueue.ts`) и сериализованная
  очередь записи с дедупликацией на сервере (`queue.js`).

### HTTP API

| Метод | Путь                  | Назначение                              |
|-------|-----------------------|-----------------------------------------|
| GET   | `/api/items`          | левое окно (`search`, `offset`, `limit`)|
| GET   | `/api/selected`       | правое окно (серверная пагинация)       |
| GET   | `/api/selected/order` | полный порядок выбранных                 |
| GET   | `/api/state`          | снимок состояния                         |
| POST  | `/api/mutations`      | батч `select` / `deselect` / `setOrder` |
| POST  | `/api/add`            | батч добавления новых ID                 |

## Запуск локально

```bash
npm install        # ставит зависимости server и client
npm run build      # собирает клиент в client/dist
npm start          # Express на http://localhost:3001 (отдаёт и API, и клиент)
```

Режим разработки (два процесса, Vite с прокси на API):

```bash
npm run dev:server   # http://localhost:3001
npm run dev:client   # http://localhost:5173
```

## Деплой

- **Render**: репозиторий + `render.yaml` (build `npm install && npm run build`,
  start `npm start`).
- **Docker**: `docker build -t fullstack-test . && docker run -p 3001:3001 fullstack-test`.

Порт берётся из переменной окружения `PORT`.
